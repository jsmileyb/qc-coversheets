## Summary
- Restricted external user role assignments to `user` and `reviewer` only, with validation and UI messaging for blocked changes. 
- Added personalized welcome messaging for internal users, with fallback behavior and a generic message for external users. 
- Added database backup/export and restore scripts, plus README documentation for usage and options. 
- Enhanced the admin template manager with JSON import/export, validation, confirmation, and optional publish flow. 

## Changes Included

### Breaking Changes
- External users can no longer be promoted to `admin` or `internal_readonly`; assignments are limited to `user` and `reviewer`. 

### Features / Enhancements
- Added a welcome line below dev page titles showing `Welcome, [FIRST NAME]` for internal users. 
- Dev pages now prefer the Entra display name and fall back to the email prefix when needed. 
- External users now see a generic welcome message. 
- Added database backup export script to dump schema tables to CSV with a manifest. 
- Added database restore script to load CSV backups with optional truncate support. 
- Added README usage examples and documented available command-line options. 
- Added file-based JSON import/export in the dev template manager. 
- Added post-import confirmation to publish the schema as a new active version. 
- Added success and error banners for template operations. 
- Updated template editor save flow to support both manual saves and imported schema publishing. 

### Fixes
- Added User Access Admin error banner for failed role updates. 

## Testing
